# Vanilla JS → Vue 3 移行ガイド（家計簿アプリ）

このガイドは、現在の `frontend/` ディレクトリにある Vanilla JS 実装を、Vue 3 (Vite + JavaScript) に移行するための手順書です。
完成コードは記載しません。各フェーズの方針と、既存コードのどこをどう置き換えるかの対応関係を示します。

---

## 1. 準備フェーズ：環境構築とファイルの引っ越し

### 1-1. Vue プロジェクトの作成

`npm create vite@latest` で **Vue** + **JavaScript** テンプレートを選択し、既存の `frontend/` とは別のディレクトリに作成する。
既存の `frontend/` は移行完了まで削除しない。

### 1-2. 既存アセットの移動

| 移動元（現在）            | 移動先（Vue）               | 備考         |
| ------------------------- | --------------------------- | ------------ |
| `frontend/styleshsst.css` | `src/assets/styleshsst.css` | 内容変更なし |
| `frontend/鉛筆.png`       | `src/assets/鉛筆.png`       | 内容変更なし |
| `frontend/.env`           | プロジェクトルートの `.env` | 内容変更なし |

### 1-3. エントリポイントの設定

- `src/main.js` で既存の CSS をインポートし、見た目を維持する
- Vite が生成する `index.html` の `<title>` と `<meta>` を既存の `frontend/index.html` に合わせる

### 1-4. API モジュールの移植

`frontend/js/models/API.js` を `src/api/API.js` としてコピーする。
`import.meta.env.VITE_API_BASE_URL` は Vite の環境変数としてそのまま動作するため、変更不要。

## 2. 転換フェーズ：DOM 操作を Vue の構文へ

既存コードの各処理を、どの Vue の機能で置き換えるかの対応表。

### 画面遷移

| 既存の実装                                          | 対応するファイル | Vue での置き換え方針                                                    |
| --------------------------------------------------- | ---------------- | ----------------------------------------------------------------------- |
| `window.navigate` 関数で `app.innerHTML` を書き換え | `app.js`         | `ref` で現在のページ名を管理し、`v-if` でコンポーネントを切り替える     |
| `index.html` の `onclick="navigate('...')"`         | `index.html`     | `App.vue` の `<template>` に移し、`@click` でリアクティブ変数を更新する |

### フォーム入力・送信

| 既存の実装                                 | 対応するファイル               | Vue での置き換え方針                                     |
| ------------------------------------------ | ------------------------------ | -------------------------------------------------------- |
| `new FormData(form)` で値を取得            | `app.js` の `handleFormSubmit` | `ref` でフォームの各値を管理し、`v-model` でバインドする |
| `e.preventDefault()`                       | `app.js` の `handleFormSubmit` | `@submit.prevent` ディレクティブに置き換える             |
| `submitBtn.disabled = true` で二重送信防止 | `app.js` の `handleFormSubmit` | `ref` でフラグを管理し、`:disabled` でバインドする       |
| `form.reset()` でフォームクリア            | `app.js` の `handleFormSubmit` | `ref` のオブジェクトを初期値に戻す                       |

### フィードバック表示

| 既存の実装                                                                  | 対応するファイル           | Vue での置き換え方針                                                                          |
| --------------------------------------------------------------------------- | -------------------------- | --------------------------------------------------------------------------------------------- |
| `form.querySelector('.form-feedback')` でDOM取得→テキスト・クラスを書き換え | `app.js` の `showFeedback` | `ref` でメッセージとエラーフラグを管理し、テンプレート側で `{{ }}` と `:class` でバインドする |
| `setTimeout` でメッセージ消去                                               | `app.js` の `showFeedback` | ロジックはそのまま。`ref` の値を空にすればテンプレートが自動更新される                        |

### カテゴリー取得・表示

| 既存の実装                                                            | 対応するファイル                        | Vue での置き換え方針                                                    |
| --------------------------------------------------------------------- | --------------------------------------- | ----------------------------------------------------------------------- |
| `getCategories()` → `filter()` で支出/収入に振り分け                  | `app.js` の `setupInputPage`            | `onMounted` でカテゴリーを取得し、`computed` で支出用・収入用に絞り込む |
| `document.querySelector(selector)` → `innerHTML` で `<option>` を生成 | `input_screen.js` の `updateCategories` | `v-for` で `<option>` を自動生成する。DOM操作は不要になる               |

### タブ切り替え

| 既存の実装                                             | 対応するファイル                     | Vue での置き換え方針                                                                             |
| ------------------------------------------------------ | ------------------------------------ | ------------------------------------------------------------------------------------------------ |
| CSS ラジオボタン（`#expense:checked ~`）で表示切り替え | `input_screen.js` + `styleshsst.css` | `ref` でアクティブタブを管理し、`v-show` で表示を切り替える。ラジオボタン用のCSSルールは削除する |

### エラー表示

| 既存の実装                                            | 対応するファイル             | Vue での置き換え方針                                              |
| ----------------------------------------------------- | ---------------------------- | ----------------------------------------------------------------- |
| `document.createElement('div')` でエラーDOMを動的生成 | `app.js` の `setupInputPage` | `ref` でエラーフラグを管理し、`v-if` でエラーメッセージを表示する |

### イベント委譲

| 既存の実装                                                                      | 対応するファイル         | Vue での置き換え方針                                                                    |
| ------------------------------------------------------------------------------- | ------------------------ | --------------------------------------------------------------------------------------- |
| `app.addEventListener('submit')` で全フォームを一括監視し、`e.target.id` で分岐 | `app.js` の `initEvents` | 各コンポーネントの `<form>` に直接 `@submit.prevent` を付ける。イベント委譲は不要になる |

## 4. 仕上げフェーズ：Vue の便利機能の導入

このフェーズは「動くようになった後」に取り組む改善です。

### 4-1. タブ切り替えの改善

CSSラジオボタンによるタブ制御を `ref` + `v-show` に置き換えたら、不要になったCSS（`#expense:checked ~` 系のルール）を `styleshsst.css` から削除する。
タブのアクティブ状態のスタイルは `:class` バインディングで制御する。

### 4-2. computed によるカテゴリー絞り込み

`categories.filter(c => c.type === 'expense')` のような処理は `computed` に移すことで、元データが更新されると自動的に再計算される。

### 4-3. v-if によるエラー表示

`document.createElement` でDOMを動的生成する代わりに、`ref` のフラグと `v-if` で宣言的にエラー表示を制御する。

### ブランチ構成

`feature/PF-21` を親ブランチとし、各作業ブランチはここから切る。

```
main
 └── feature/PF-21（親ブランチ：各PRのマージ先）
      ├── feature/PF-21-vue-setup
      ├── feature/PF-21-input-screen
      ├── feature/PF-21-result-screen
      └── feature/PF-21-cleanup
```

### PR 一覧

| PR  | ブランチ名                    | マージ先        | 内容                                                                  | 完了条件                              |
| --- | ----------------------------- | --------------- | --------------------------------------------------------------------- | ------------------------------------- |
| #1  | `feature/PF-21-vue-setup`     | `feature/PF-21` | Vueプロジェクト作成、CSS・画像・`API.js` の移植、`App.vue` のナビ部分 | アプリが起動しナビボタンが表示される  |
| #2  | `feature/PF-21-input-screen`  | `feature/PF-21` | `InputScreen.vue`（タブ・フォーム・送信・カテゴリー取得）             | 収支の入力・保存が一通り動く          |
| #3  | `feature/PF-21-result-screen` | `feature/PF-21` | `ResultScreen.vue` の実装                                             | 収支一覧が表示される                  |
| #4  | `feature/PF-21-cleanup`       | `feature/PF-21` | 旧 `frontend/` の削除、`docker-compose.yml` のパス更新                | 旧コードが除去され、Vueのみで動作する |

すべてのPRが `feature/PF-21` にマージされた後、`feature/PF-21` を `main` にマージする。

### 作業手順

```bash
# 親ブランチを作成
git switch main
git switch -c feature/PF-21

# PR #1 の作業ブランチ(feature/PF-21からブランチを切る)
git switch -c feature/PF-21-vue-setup

# PR #1 を feature/PF-21 にマージ後、PR #2 を切る
git switch feature/PF-21
git pull
git switch -c feature/PF-21-input-screen

# 以降も同様
```

### 分割の考え方

- **「その画面が動く」単位で切る**。各PRは動作確認できる状態でマージする
- 前のPRをマージしてから次のブランチを切ることでコンフリクトを防ぐ
- 旧 `frontend/` の削除は最後の PR #4 にまとめる。途中で消すと切り戻しが困難になるため

## 7. ファイル名の修正

移行のタイミングで、Vue / JavaScript の慣習に合わないファイル名を修正してください。

| 現在のファイル名 | 問題点                                                                               | 推奨するファイル名                   |
| ---------------- | ------------------------------------------------------------------------------------ | ------------------------------------ |
| `API.js`         | JS のモジュールファイルは **camelCase** が一般的。全大文字はクラスや定数と紛らわしい | `api.js`                             |
| `styleshsst.css` | `stylesheet` のタイポと思われる。移行を機に用途が分かる名前にする                    | `global.css` など                    |
| `鉛筆.png`       | 日本語ファイル名はOS・ツール間で文字化けやパス解決の問題を起こしやすい               | `pencil.png` や `icon-edit.png` など |

> **補足**: Vue コンポーネント（`.vue`）は **PascalCase**（`InputScreen.vue`）が公式の推奨です。
> 今回のガイドで示しているコンポーネント名はこの慣習に沿っているため変更不要です。

ファイル名を変更した場合は、`import` パスや `<img src="...">` の参照も忘れずに更新してください。
